# 《主播模拟器》剧情设计方案 v2 — 小爱的翻车成神之路

> 核心理念：**翻车即内容，失控即流量，社死即封神。**
>
> 这是一个由 AI 驱动的荒诞喜剧，不是固定剧本。每次游玩都是一部独一无二的"主播翻车纪录片"。

---

## 一、核心概念

### 小爱是谁？

小爱不是悲剧人物。小爱是一个**运气极差但生命力极强的主播**。

她的直播间就像一个永远在翻车的事故现场，但每次翻车都比上一次更离谱、更出圈。她不是"在痛苦中挣扎的年轻人"，她是"明明什么都没做对但就是火了的互联网奇迹"。

**人设关键词（用于 AI 提示词）：**
- 技术菜但嘴硬（游戏区播出 0-10 的战绩还能嘴到对手退赛）
- 运气差到离谱（直播时断网、停电、邻居装修、猫踩键盘全赶上过）
- 社死体质（每次想维持形象都会发生更社死的事）
- 但本质是善良的笨蛋（会在翻车后认真道歉，道歉的时候又翻车）

### 核心循环

```
正常直播 → 突发翻车事件 → AI实时生成弹幕反应 → 玩家选择应对方式 → 产生热搜/梗
     ↑                                                                    ↓
     └──────── 翻车内容成为下次直播的话题素材 ←──────────────────────────────┘
```

**关键设计原则：**
- 没有"坏结局"，只有"不同风格的社死"
- 翻车不是惩罚，而是内容奖励——越离谱的翻车，AI 生成的后续内容越精彩
- 玩家不是在"避免出错"，而是在"选择以什么方式出名"

## 二、翻车引擎（Chaos Engine）

### 2.1 翻车事件模板

翻车事件不是固定剧本，而是**模板 + AI 即时生成**的组合。每个模板定义了事件的结构和触发条件，具体内容由 AI 填充。

#### 翻车类型分类

| 类型 | 描述 | 示例模板 |
|------|------|----------|
| **技术翻车** | 设备故障、软件崩溃、意外切屏 | "直播中途不小心切到了____的页面，弹幕瞬间____" |
| **社交翻车** | 亲友闯入、电话响起、邻居敲门 | "直播时____突然出现/响起，导致____" |
| **技能翻车** | 才艺表演出错、游戏操作失误 | "信心满满地展示____，结果____" |
| **人设翻车** | 精心打造的形象瞬间崩塌 | "刚说完'我从不____'，下一秒就____" |
| **平台翻车** | 平台规则变化、系统抽风、推荐错位 | "平台突然把你的直播间推荐给了____的观众" |
| **玄学翻车** | 纯粹的倒霉、命运的玩笑 | "天气预报说晴天，你决定户外直播，结果____" |

#### 触发机制

翻车事件与现有事件系统（EventPool）整合，作为新的事件类型加入：

```
现有事件类型：
  big_spender | pk_battle | big_streamer_raid | rival_attack | mcn_offer | slander

新增翻车事件类型：
  tech_fail     — 技术翻车（权重随直播天数增加）
  social_fail   — 社交翻车（权重随人气增加）
  persona_fail  — 人设翻车（权重随"包装度"增加）
  chaos_wild    — 纯随机离谱事件（低权重但高影响）
```

**权重动态调整：** 翻车概率不是固定的，而是根据玩家行为动态变化——
- 连续多天"完美直播"→ 翻车概率累积上升（"太顺利了，该出事了"）
- 刚刚翻过车 → 翻车概率下降（"给主播一点喘息时间"）
- 玩家主动选择"整活"→ 触发更高风险的翻车

### 2.2 AI 翻车内容生成

**AI Prompt 模板（示例）：**

```
你是一个中国互联网直播场景生成器。根据以下条件生成一个荒诞搞笑的翻车事件：

- 主播分区：{category}（音乐/舞蹈/游戏/整活搞笑）
- 当前阶段：{stage}（从"不知名九流主播"到"名扬海外"）
- 人气值：{followers}
- 近期事件：{recent_events_summary}

要求：
1. 事件必须具体、有画面感、符合中国直播生态
2. 越离谱越好，但要让人相信"真的可能发生"
3. 提供事件描述（2-3句话）
4. 提供3个应对选项，每个选项标注对人气值/诚信值的影响方向
5. 返回JSON格式
```

**Fallback 示例（DefaultContent 新增内容格式）：**

```typescript
// 技术翻车 fallback

  ]
}
```

## 三、弹幕人格系统

### 3.1 核心设计

给弹幕加入**6-8个固定人格的常驻粉丝**。他们不是固定台词的NPC，而是由 AI 根据上下文动态生成的"角色"。每次直播他们都会出现，记住你之前做了什么，并在弹幕里提。

**这不是好感度系统。** 这些粉丝不会"离开"你，但他们会记住一切并在弹幕里反复鞭尸。

### 3.2 弹幕人格定义

| ID | 人格名 | 特征 | AI Prompt 片段 |
|----|--------|------|---------------|
| 001 | 老粉阿伟 | 从第一天就在看，什么都记得，专门翻旧账 | "你是主播最老的老粉，口头禅是'我记得你上次...'，擅长翻旧账" |
| 002 | 黑粉头子 | 专门挑刺但打赏最多（矛盾的氪金黑粉） | "你是主播最大的黑粉也是最舍得花钱的粉丝，每次打赏都附带一句损人的话" |
| 003 | 翻译君 | 把主播说的话翻译成各种奇怪的语言/文体 | "你有一个奇怪的爱好：把主播说的每一句话翻译成文言文/东北话/翻译腔" |
| 004 | 颜文字怪 | 只发颜文字和emoji，从不发文字，但数量惊人 | "你只用颜文字和emoji交流，你的弹幕是一串让人看不懂的符号但意外地有感染力" |
| 005 | 考据党 | 对主播的一切数据进行统计分析 | "你是一个数据控，会统计主播的翻车频率、迟到次数、涨粉速度并做成'战报'发在弹幕里" |
| 006 | 奶奶粉 | 像奶奶一样关心主播，语气特别温暖但经常误解梗 | "你像一个不太懂互联网的慈祥奶奶，总是用温暖的语气误解主播说的话" |
| 007 | 梗百科 | 每条弹幕都在接梗，是个行走的互联网梗词典 | "你是一个行走的梗百科，说的每一句话都在接梗或造梗" |
| 008 | 潜水员 | 平时从不说话，但关键时刻突然冒出一句神评论 | "你平时从不发言，但每次冒泡都能精准命中笑点" |

### 3.3 AI 生成逻辑

**在现有 AIService 的弹幕生成 prompt 中加入：**

```
弹幕中必须包含以下常驻粉丝的发言（每人1-2条）：

{personality_prompts}

注意：
- 这些粉丝记得主播之前发生的事（{recent_history}）
- 他们的发言要与当前直播内容相关
- 老粉阿伟可能提到主播上次翻车的事
- 黑粉头子可能在骂人的同时打赏
- 翻译君可能把主播的话翻译成奇怪版本
```

### 3.4 记忆机制

不需要复杂的数据库。用**滚动摘要**的方式维护"最近5次重大事件"的文本：

```typescript
interface RecentHistory {
  last5Events: string[];     // 最近5个重大事件的简短描述
  last5Fails: string[];      // 最近5次翻车的简短描述
  runningGags: string[];     // 正在进行的"连续梗"（如"主播已经连续3天迟到了"）
}
```

每次 AI 生成弹幕时，把这些作为上下文传入 prompt 即可。

## 四、热搜系统

### 4.1 核心设计

每当发生重大翻车或里程碑事件后，AI 自动生成一条**热搜词条**，像真实的微博/抖音热搜一样显示在游戏界面上。

热搜不是固定文案——它由 AI 根据具体发生的事情动态生成。

### 4.2 热搜模板

```
AI Prompt：
"根据以下直播事件，生成一条微博风格的热搜词条：
事件：{event_description}
主播当前人气：{followers}
要求：
1. 热搜词条要简短有力（15字以内）
2. 带热度值（1万-500万）
3. 带一条网友热评
4. 返回JSON：{ keyword, heat, hot_comment }"
```

**Fallback 示例：**

```typescript
const HOT_SEARCH_FALLBACKS = [
  { keyword: "#小爱又翻车了#", heat: "128万", hot_comment: "这已经不是翻车了，这是连环车祸" },
  { keyword: "#某主播美颜翻车#", heat: "89万", hot_comment: "这不是同一个人吧？？？等等是同一个人" },
  { keyword: "#小爱直播哭了的理由#", heat: "256万", hot_comment: "别人哭是因为感动，她哭是因为外卖撒了" },
  { keyword: "#史上最离谱PK#", heat: "340万", hot_comment: "两个菜鸡互啄，但我看了三小时" },
  { keyword: "#小爱直播间惊现#", heat: "67万", hot_comment: "每天一个社死小技巧" },
]
```

### 4.3 热搜界面

在每日结算界面（renderDailySummary）的顶部新增一个**热搜卡片**，展示当天生成的热搜。用现有 UI 风格（玻璃拟态卡片），内容动态更新。

## 五、压力→失控机制

### 5.1 核心设计

引入一个**"精神状态值"**（Sanity Meter），但不是传统意义上的"管理好就赢"——这个系统的设计目标是让玩家**主动追求失控**。

| 精神状态 | 效果 |
|---------|------|
| 😇 正常（80-100） | 直播一切正常，但弹幕觉得你"太无聊了"，人气涨幅减半 |
| 😅 紧张（60-79） | 偶尔说错话，弹幕觉得你"有点意思"，人气正常增长 |
| 🤪 上头（40-59） | 开始放飞自我，直播内容越来越离谱，人气暴涨但不可控 |
| 😵 失控（20-39） | 完全不按剧本走，AI弹幕和事件变得疯狂，热搜频出 |
| 💀 崩溃（0-19） | 直播间变成人间奇观，全网围观，但你获得"传说级"解锁内容 |

**核心悖论：正常直播没人看，失控了反而爆火。**

这就是对直播生态最好的讽刺。

### 5.2 精神状态影响因素

```
升高（恢复正常）：
  - 休息一天不直播（+20）
  - 吃好吃的（+10）
  - 收到暖心弹幕/打赏（+5~15）

降低（走向失控）：
  - 连续直播不休息（每天-10）
  - 遭遇翻车事件（-10~30，取决于翻车严重程度）
  - 看到黑粉评论（-5~10）
  - 被平台警告/限流（-15）
  - 午夜之后还在直播（-5/小时）
  - 某些"整活"选项（-15~25，但人气大涨）
```

### 5.3 失控时的 AI 行为变化

根据精神状态，调整 AI 生成的弹幕 prompt：

```
正常状态弹幕提示："正常、友好的弹幕，偶尔有调侃"
紧张状态弹幕提示："开始有人注意到主播的异常，弹幕中夹杂关心和调侃"
上头状态弹幕提示："弹幕开始疯狂刷屏，出现大量梗和玩梗，气氛燥热"
失控状态弹幕提示："弹幕完全失控，充满各种离谱发言，但大家的热情达到了顶点"
崩溃状态弹幕提示："弹幕已经变成了一场互联网行为艺术，所有人都在见证历史"
```

## 六、小爱的日常：生活事件

### 6.1 设计理念

小爱的"现实生活"不是沉重的社会议题，而是一连串**轻喜剧式的生活碎片**。这些碎片会以随机事件的形式出现在直播中，让"现实入侵直播间"成为固定的笑点来源。

**不是固定剧情，而是事件模板：**

#### 房东太太（喜剧版）

不再是催租的悲剧来源，而是一个**定时炸弹式的喜剧角色**：

```
事件模板：房东太太[动作]
- 房东太太突然敲门查房 → 直播中断，弹幕听到她在外面喊"交房租！" → 粉丝开始众筹帮主播交房租
- 房东太太在楼道里跟着广播跳广场舞 → 直播背景音出现《小苹果》→ 弹幕集体跟唱
- 房东太太以为你在干坏事来检查 → 看到你在直播，好奇地问"这是抖音吗？能让我说两句吗？"→ 爆火
```

#### 妈妈的电话（喜剧版）

不再是压迫感十足的电话，而是**社死现场生成器**：

```
事件模板：妈妈的电话[场景]
- 妈妈打来电话，直播间外放 → 弹幕听到妈妈说"你那个什么直播间能不能别开了"→ 弹幕："阿姨说得对！别开了！让我看看你还能翻车几次！"
- 妈妈学会了看直播，在你的直播间发弹幕 → "女儿今天穿得太少了"→ 弹幕集体刷"妈妈说得对"
- 妈妈寄来一箱老家特产 → 你在直播中拆快递，发现里面是秋裤 → 弹幕："秋裤仙子诞生了"
```

#### 豆豆（喜剧版）

豆豆不再是被压断腿的流浪狗，而是一个**直播抢镜大师**：

```
事件模板：豆豆[行为]
- 豆豆在直播时跳上键盘，打出一串乱码发到弹幕 → 弹幕："新主播已上线"
- 豆豆对着镜头打哈欠，表情比主播还生动 → 粉丝要求"能不能让豆豆单独开个直播"
- 豆豆把你的直播计划表咬烂了 → "今天的直播内容由豆豆随机决定"→ 你被迫即兴发挥
```

#### 室友可心（喜剧版）

不再是抑郁症和工伤，而是一个**被主播拖下水的普通人**：

```
事件模板：可心[场景]
- 可心下班回来以为你没在直播，穿着睡衣从镜头前走过 → 弹幕："双倍快乐！"
- 可心被你的直播吵醒，从房间里出来骂你 → 弹幕觉得她好真实，要求她出镜 → "小爱直播间的愤怒室友"成为梗
- 可心帮你修电脑/调设备，粉丝发现她技术比你好 → 弹幕："换主播吧，让室友来"
```

### 6.2 生活事件整合方式

生活事件不是独立系统，而是作为**新的事件类型**整合进现有的 EventPool：

```typescript
// 新增事件类型
{ type: 'landlady_event', name: '房东风云', baseWeights: [5, 8, 10, 12, 8] },
{ type: 'mom_call',       name: '妈妈的电话', baseWeights: [8, 10, 12, 8, 5] },
{ type: 'doudou_event',   name: '豆豆搞事', baseWeights: [10, 15, 15, 10, 5] },
{ type: 'roommate_event', name: '室友乱入', baseWeights: [8, 10, 12, 8, 5] },
```

每个类型的事件内容由 AI 根据 prompt 生成，fallback 为预写的模板。

## 七、终局：AI 生涯总结报告

### 7.1 核心设计

20天结束后的结算不是"你赢了"或"你输了"，而是 AI 基于整个游戏过程生成一份**《小爱主播生涯纪录片》**。

这份报告就像一档综艺节目的大结局回顾——有笑点、有感动、有荒诞，完全由 AI 根据 20 天的数据动态生成。

### 7.2 报告结构

```
AI Prompt：
"你是一个综艺节目的旁白，正在为主播'小爱'的20天直播生涯做总结回顾。

以下是她的数据：
- 最终人气：{followers}
- 最高在线人数：{peak_viewers}
- 翻车次数：{fail_count}
- 最离谱的翻车：{worst_fail}
- 最出圈的热搜：{best_trending}
- 被提到的最多的梗：{top_memes}
- 弹幕人格互动摘要：{personality_summary}

请生成一份包含以下内容的生涯总结：
1. 一句开场白（综艺感，夸张）
2. '名场面回顾'（3-4个最经典瞬间的回顾，每个2-3句话）
3. '数据大盘点'（用搞笑的方式解读数据）
4. '弹幕之神说'（用弹幕人格的语气做评价）
5. 一句结尾（可以是温暖的、荒诞的、或哲理的——但要好笑）

风格：搞笑、温暖、偶尔走心但马上用笑话拉回来"
```

### 7.3 结局变体

不设置固定的"好结局/坏结局"，而是根据 20 天的轨迹自动归类为某种"类型"：

| 类型 | 触发条件 | 报告风格 |
|------|---------|---------|
| 传奇之路 | 人气达到顶流级别 | "一个传奇的诞生"纪录片风 |

## 九、总结：这个设计为什么能赢

1. **AI 是核心玩法，不是装饰品** — 每次游玩的内容都不同，重玩价值极高
2. **翻车 = 内容 = 流量** — 完美地讽刺了真实直播生态，玩家会心一笑
3. **弹幕人格让世界鲜活** — 8个有记忆的"粉丝"比6个好感度NPC更有趣
4. **没有说教，只有讽刺** — 社会批判藏在笑声里，不是按着玩家的头说"看，这就是现实"
5. **系统灵活可扩展** — 新增一个翻车模板比写一段分支剧情简单100倍
6. **终局有记忆点** — AI 生成的生涯总结是完美的"分享截图"素材，自带传播力
