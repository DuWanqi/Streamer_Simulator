# 主播模拟器（双面人生）— 完整项目架构文档

> 文档生成日期：2026-04-20
> 基于当前 main 分支代码（V0.1 版本，含双面人生剧情系统 + 桌面模拟）

---

## 一、项目概况

| 项 | 值 |
|----|-----|
| 类型 | AI原生模拟经营游戏 |
| 技术栈 | TypeScript + Vite 5 + PixiJS v8 |
| 主语言 | TypeScript（strict mode, ES2020, ESNext modules） |
| UI方案 | DOM渲染（innerHTML模板字符串），非 PixiJS 画布 |
| AI模型 | Google Gemini 2.0 Flash |
| 样式 | 内联 style + Tailwind CDN + 3个独立CSS文件 |
| 构建工具 | Vite（dev端口 5179） |
| 测试 | 无测试框架，仅 `tsc` 类型检查 |
| 代码量 | ~10,000行 TypeScript |

**所有源码在 `2d_game_template/src/` 下，所有命令在 `2d_game_template/` 下执行。**

### 4.2 直播场景详细流程

```
renderLivestream()
│
├─ 初始化直播UI（观众数、弹幕区、评论区、事件区）
├─ 启动弹幕定时器（800ms/条）
├─ 启动评论定时器（2000ms/条）
├─ 生成今日事件列表：eventPool.generateDayEvents(state) → 3-5个
│
├─ 事件依次触发（每个间隔3-5秒）：
│  ├─ 普通事件：弹窗显示事件描述 + 选项/确认
│  └─ PK事件：renderPKBattle() → 13秒倒计时点击小游戏
│     ├─ 准备阶段（3秒）：显示对手信息
│     └─ 对战阶段（10秒）：疯狂点击，比较点击数
│        ├─ 胜利：+200收入，+粉丝
│        └─ 失败：收入不变
│
├─ 所有事件结束 或 玩家点击"结束直播"
│  ├─ clearInterval（弹幕+评论定时器）
│  └─ changeScene('daily_summary')
│
└─ ⚠ 已知问题：如果用户未点"结束直播"就导航，定时器不会清除
```

### 4.5 每日流程分支（handleDoubleLifeDailyFlow）

这是"下一天"按钮触发后的完整逻辑，**决定了新一天的内容**：

```
handleDoubleLifeDailyFlow()
│
├─ 1. 生存系统处理：survivalSystem.processDaily()
│     扣除日常开销（房租100+水电20+食物30+网费10=160元/天）
│     优先级：网费 > 食物 > 水电 > 房租
│     不足时降低精神状态+粉丝惩罚
│
├─ 2. NPC日常交互检查：npcSystem.checkDailyInteraction()
│     首次出现日强制触发 + 20%概率随机触发
│     └─ 如果触发 → 显示NPC对话 → ⚠ 直接return，跳过直播
│
├─ 3. 主线剧情节点检查：getStoryNodeByDay(day, state)
│     按node1→node6顺序检查：
│     ├─ 当前天数在触发范围内？
│     ├─ 未触发过？
│     ├─ 前置条件满足？
│     └─ 如果匹配 → showStoryNode() → 显示剧情+选项+应用效果
│        ├─ 70%概率生成热搜
│        └─ ⚠ 直接return，跳过直播
│
├─ 4. 随机事件检查：30%概率
│     getRandomEvent() → showRandomEvent()
│     └─ ⚠ 直接return，跳过直播
│
└─ 5. 正常流程
      changeScene('stream_planning')
      → main_hub → livestream → daily_summary → 循环
```

**⚠ 关键问题：** 步骤2/3/4如果触发，玩家当天无法直播、无法升级。意味着部分天数完全没有核心玩法。

## 五、剧情系统详解

### 5.1 六个主线节点（StoryNodes.ts）

| 节点 | 名称 | 触发天数 | 前置条件 | 选项数 |
|------|------|---------|---------|--------|
| node1 | 房东太太晕倒 | Day 3-5 | 无 | 4 |
| node2 | 邻居骚扰与网暴 | Day 11-13 | 粉丝>10000 | 4 |
| node3 | 捡到豆豆 | Day 6 | 无 | 4 |
| node4 | 可心的悲剧 | Day 10-12 | 可心好感>50 | 4 |
| node5 | 世纪PK | Day 16-18 | 粉丝>500000 | 4 |
| node6 | 妈妈的阿尔茨海默 | Day 20 | 无 | 4 |

**每个节点的数据结构：**
```
StoryNode {
  id: string               // 如 "node1_landlady"
  title: string            // 节点标题
  triggerDay: [min, max]   // 触发天数范围
  emotion: string          // 角色表情（happy/nervous/angry/...）
  monologue: string        // 小爱内心独白
  sceneDescription: string // 场景描述
  dialog: string           // NPC对话
  choices: EventChoice[]   // 选项列表
    └─ { text, effects: { followers?, kindness?, integrity?, sanity?,
                         money?, npcRelation?, personaIntegrity? } }
  isMajor: boolean         // 是否可能触发热搜
  prerequisite?: function  // 前置条件检查
}
```

### 5.2 三十个随机事件（RandomEvents.ts）

5个分类，每类6个：

| 分类 | 事件示例 |
|------|---------|
| 技术翻车（tech） | 美颜滤镜崩溃、忘记关麦、软件崩溃、直播睡着、碰倒摄像头、网络卡顿 |
| 社交翻车（social） | 念错粉丝ID、PK对手嘲讽、粉丝表白、被路人认出、黑粉攻击、土豪打赏 |

## 六、子系统详解

### 6.1 NPC系统（NPCSystem.ts）

- **首次登场**：6个NPC各自在固定天数首次出现，强制触发对话
- **日常互动**：20%概率触发随机NPC互动（房东做饭、可心吐槽、豆豆玩耍等）
- **对话**：首次对话有3-4个选项影响好感度；日常互动为固定文本
- **好感度**：0-100范围，影响结局判定和特定事件解锁
- **头像**：目前使用emoji代替（⚠ 未接入实际立绘）

### 6.2 生存系统（SurvivalSystem.ts）

每日固定开销：
| 项目 | 金额/天 | 欠款后果 |
|------|--------|---------|
| 房租 | 100元 | 欠款过多→被赶出→游戏失败 |
| 水电 | 20元 | 无法使用直播设备 |
| 食物 | 30元 | 精神状态下降 |
| 网费 | 10元 | 无法直播 |

支付优先级（钱不够时）：网费 > 食物 > 水电 > 房租

可通过微信模拟APP手动交费。

### 6.3 弹幕系统（DanmakuSystem.ts）

8个常驻弹幕人格，每个有：
- 模板消息库（带变量替换 `{count}`, `{followers}` 等）
- 事件反应模板（翻车/成功/PK胜/PK败）
- 人格专属颜色
- 由 DanmakuOverlay.ts 渲染为滚动弹幕（5轨道，随机速度6-10秒）

### 6.4 热搜系统（HotSearchSystem.ts）

- 11个事件类型的关键词模板
- 热度值：指数分布，10K-5M
- 热评模板
- 主线节点70%概率触发热搜
- 可在微博APP的热搜Tab查看历史

### 6.5 结局系统（EndingSystem.ts）

| 结局 | 条件 |
|------|------|
| reconciliation（自我和解） | 善良≥60, 诚信≥60, 妈妈≥60, 豆豆≥60 |
| lost（虚拟迷失） | 善良<30 或 诚信<30 |
| escape（回归现实） | 精神状态<20 |
| collapse（双重崩塌） | 破产/被赶出 且 善良<20 |
| redemption（真相救赎） | 善良≥70, 诚信≥70, 所有NPC≥60, 特定妈妈选择 |

每个结局生成：标题、描述文案、真相揭示（小爱停播原因）、尾声

### 6.6 生涯总结系统（CareerSummarySystem.ts）

6种总结类型：传奇之路、翻车之王、疯美人、真实自我、社死传说、佛系

每个包含：开场白、名场面回顾、数据盘点、弹幕人格评价、真相、结尾

可生成 AI Prompt 交由 Gemini 动态生成。

### 6.7 技能树系统（SkillTreeConfig.ts）

4个维度，每维度3层（基础/进阶/终极），共24个技能节点：

| 维度 | 示例技能 |
|------|---------|
| 口才 | 应急口胡修复、黑粉护盾、话术大师 |
| 外貌 | 镜头美化术、穿搭升级、流量颜值王 |
| 才艺 | 即兴表演、才艺精进、全能艺人 |
| 知识 | 热点嗅觉、内容策划、知识大V |

每层需要前一层解锁 + 消耗技能点。

### 8.1 AI调用点

| 功能 | 调用时机 | Prompt风格 |
|------|---------|-----------|

`cachedCall(key, fn)` 模式，每日首次调用后缓存结果。
