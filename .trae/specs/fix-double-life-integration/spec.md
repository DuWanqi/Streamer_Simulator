# Fix Double Life Integration Spec

## Why

当前游戏存在严重问题：
1. 开局直接显示分区选择，缺少穿越剧情开场
2. 半身像显示为emoji而非实际图片
3. NPC对话系统未接入游戏流程
4. 生存支付系统完全不可见
5. 所有新系统没有实际融入游戏流程

需要按照设计案 `d:\projects\Streamer_Simulator\.trae\剧情策划方案_完整版.md` 重新整合所有系统。

## What Changes

### 新增/修改内容

1. **开场穿越剧情**
   - 新增开场场景：玩家作为粉丝观看停播100天后的小爱直播
   - 穿越动画：突然穿越回第1天，绑定林小爱身份
   - 机械音引导："请走完小爱的人生之路，找到她停播100天的真相"
   - 使用完整立绘（白毛1.3.png）作为开场背景

2. **对话UI系统重构**
   - 横板对话布局：主角立绘在右侧，NPC立绘在左侧
   - 对话框在底部，支持多行文本
   - 9种表情立绘根据情绪自动切换
   - 图片路径正确配置，确保能加载

3. **游戏流程重新整合**
   - 每日流程：开场独白 → 生存检查 → NPC互动 → 直播 → 事件 → 结算
   - 第1天强制触发开场剧情
   - 特定天数强制触发剧情节点（第3-5天房东事件等）

4. **NPC对话系统接入**
   - 在对应节点显示NPC对话
   - 房东太太：第1天、第3-5天
   - 可心：第3天首次出现
   - 豆豆：第6天捡到
   - 猥琐男：第11天
   - 月牙儿：第16天
   - 妈妈：第20天

5. **生存系统可视化**
   - 每日结算显示房租、水电、食物、网费支出
   - 拖欠状态警告显示
   - 危机事件触发时显示对话框

6. **图片路径修复**
   - 确认半身像路径：`./portraits/` 或 `/portraits/`
   - 确认完整立绘路径
   - 添加图片加载错误处理

## Impact

  - `src/game/Game.ts` - 主要修改，添加开场场景和流程整合

## ADDED Requirements

### Requirement: 开场穿越剧情

**场景描述**：
玩家是小爱的忠实粉丝，停播100天后突然收到直播推送。观看时屏幕闪烁，玩家穿越回小爱刚起步的第1天。

**实现要求**：
- 使用完整立绘（白毛1.3.png）作为背景
- 分步骤显示剧情文本，有"下一步"按钮
- 机械音对话框样式
- 最后显示"绑定成功，游戏开始"

**文案**：
```
【第1幕：粉丝】
你是一名普通的大学生，也是主播"小爱"的忠实粉丝。
100天前，小爱突然停播，没有任何解释。

今天，你收到了直播推送...

【第2幕：穿越】
屏幕上，小爱的身影逐渐模糊...
一阵眩晕袭来...

【机械音】
"检测到强烈执念，启动穿越程序..."
"绑定角色：林小爱"
"任务：走完她的人生之路，找到停播的真相"

【第3幕：开始】
你睁开眼睛，发现自己坐在简陋的出租屋里。
面前是一台老旧的电脑，屏幕上显示着直播界面。

这是...小爱刚起步的第1天？
```

**布局要求**：
```
┌─────────────────────────────────────────┐
│  [NPC立绘]                    [主角立绘] │
│     左                           右      │
│                                          │
│  ┌─────────────────────────────────┐    │
│  │  对话框                          │    │
│  │  说话人名字                      │    │
│  │  对话内容...                     │    │
│  │                                  │    │
│  │  [选项1] [选项2] [选项3] [选项4] │    │
│  └─────────────────────────────────┘    │
```

**样式要求**：

- 选项按钮：横向排列，有明显hover效果
- 情绪映射：根据对话内容自动切换主角表情

## MODIFIED Requirements

### Requirement: 游戏启动流程

**原流程**：
开始界面 → 分区选择

**新流程**：
开始界面 → 开场穿越剧情 → 第1天白天（房东对话）→ 分区选择

### Requirement: 每日流程

**原流程**：
分区选择 → 直播 → 结算

**新流程**：
开场独白（带立绘） → 生存检查 → NPC互动（如有） → 分区选择 → 直播 → 事件（如有） → 热搜（如有） → 结算（含生存支出）
